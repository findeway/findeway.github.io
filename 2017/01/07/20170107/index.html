阿里云ubuntu镜像添加第三方源403的问题

发表于 2017-01-07 | | 阅读次数

为什么要写这篇小文章

平时项目需要经常用到阿里云的服务。第一次在阿里云上使用ubuntu镜像，发现一个小问题，奈何google了1小时居然没有找到答案。希望其他同学不要在这个问题上浪费自己的时间，而去做一些更加有意义的事情！

一个简单到google找不到答案的小问题

最近因为项目需要，要使用阿里云搭建一个基于kurento的media server。kurento居然限制操作系统必须使用ubuntu 14.04。kurento安装需要添加一个第三方的源，在apt-get update时，发现阿里云抛出了403错误！查看源列表内容发现阿里云的镜像已经将源列表全部替换成了阿里云的源，大大的加快了我们下载软件的速度，这还是有点小贴心～但是为什么第三方源会403呢？

于是添加了ubuntu官方的源进行测试，发现依旧403！于是怀疑是阿里云做了某些限制导致了这个问题。直接使用curl下载目标路径的文件，居然可以下载成功！看来是和apt相关的一个问题。google大法一小时居然毫无相关内容！于是给阿里云提了工单，但是技术组小哥给我的回复有点答非所问。不想傻傻地等待他们回复，于是开始自己查找原因。平时使用apt也基本都是常规使用，改改源列表，没有过多尝试过。有问题第一反应就是找配置文件，/etc/apt下面有个apt.conf，这个是什么呢？之前没有注意过。打开一看，葵花宝典！

这不就是个http代理吗？还制定了阿里云的源相同的域名，把这行代码注掉后问题迎刃而解！

写在做后

这样一件小事其实不是很值当写篇博客，自己也考虑过，没有干货；甚至很多同学分析的可能会更容易更快速的定位问题（原谅我对apt了解不多，用了不短的时间却没有仔细研究过它的配置～）。但是仔细想想大家都经常使用google搜索问题，其实很多盲目的搜索也会浪费不少时间，而这个小问题的使用场景其实也还比较容易用到的。为了填补搜索结果中的一处空白，于是写了这篇博客，希望能给其他同学提供一点帮助，节约大家的时间，把精力放在真正重要的问题上去！
